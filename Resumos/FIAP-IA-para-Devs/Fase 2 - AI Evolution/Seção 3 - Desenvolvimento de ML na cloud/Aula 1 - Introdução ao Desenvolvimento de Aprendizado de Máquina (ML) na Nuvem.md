# Aula 1 - Introdução ao Desenvolvimento de Aprendizado de Máquina (ML) na Nuvem

## Resumo executivo

Esta aula introduz o **desenvolvimento de ML na nuvem**: benefícios de **escalabilidade** (recursos sob demanda), **acesso a GPUs/TPUs** para treino pesado, **serviços gerenciados** (treino, deploy, monitoramento) e **integração** com dados e pipelines. Ciclo típico: preparação de dados, treino (local ou na nuvem), registro de modelos, deploy (endpoints, containers), monitoramento e retreinamento. Conceitos de **MLOps** e **CI/CD para modelos**; diferença entre treino local (laptop, servidor) e na nuvem (custos, tempo, capacidade).

**Objetivos de aprendizagem:** Justificar o uso da nuvem para ML (escala, GPU, serviços); descrever etapas do ciclo de vida de um modelo em produção; reconhecer serviços de ML das principais clouds.

---

## Conceitos-chave (flashcards)

1. **Por que fazer ML na nuvem?** **R:** Escalabilidade, acesso a hardware (GPU/TPU), serviços gerenciados (treino, deploy), integração com dados e pipelines.
2. **O que é MLOps?** **R:** Práticas para operacionalizar ML: versionamento de dados e modelos, pipelines de treino e deploy, monitoramento e retreinamento.
3. **Quais etapas típicas do ciclo de ML?** **R:** Dados → pré-processamento → treino → avaliação → registro do modelo → deploy → monitoramento.

---

## Perguntas para teste de reforço

1. Por que desenvolver ML na nuvem? **R:** Escalabilidade, acesso a GPU/TPU, serviços gerenciados (treino, deploy), integração com dados e pipelines.
2. O que é MLOps? **R:** Práticas para colocar ML em produção: versionamento, pipelines, monitoramento e retreinamento.
3. Cite duas etapas do ciclo de vida de um modelo. **R:** Treino, registro do modelo, deploy, monitoramento (entre outras).
4. Qual a vantagem de usar GPU na nuvem para ML? **R:** Treino mais rápido para modelos grandes; recursos sob demanda sem comprar hardware.
